# pages-uedsports-guide

## 项目简介

本仓库用于归档和发布一组独立 HTML 页面。这些页面不隶属于任何特定网站或域名，而是作为通用页面资源进行管理和版本控制。仓库的主要目的是提供结构化的页面存档，便于检索、复用与协作。

## 目录结构

```
.
├── README.md               # 本文件
├── index.html              # 首页 / 导航页
├── pages/                  # 存放所有独立 HTML 页面
│   ├── example-page1.html
│   ├── example-page2.html
│   └── ...
└── assets/                 # 页面引用的静态资源（CSS、JS、图片等）
    ├── css/
    ├── js/
    └── images/
```

- `index.html`：可选的导航或入口页面，方便浏览全部归档页面。
- `pages/`：核心目录，每个文件为一个独立页面，命名清晰，避免歧义。
- `assets/`：公共资源文件夹，供多个页面共享使用，避免重复引入。

## 页面归档说明

- 所有页面均为纯 HTML 文件，不依赖后端服务，可直接在浏览器中打开。
- 页面之间彼此独立，不强制相互链接，但可通过 `index.html` 统一索引。
- 归档页面可能包含示例数据、占位内容或通用模板，请根据实际需求修改后使用。
- 页面内引用的资源路径均采用相对路径，确保仓库克隆后可离线浏览。

## 使用方式

1. 克隆本仓库到本地：
   ```bash
   git clone https://github.com/your-username/pages-uedsports-guide.git
   ```
2. 直接在浏览器中打开 `index.html` 或 `pages/` 下的任一页面。
3. 如需部署，可将整个仓库上传至静态托管服务（如 GitHub Pages、Netlify 等），无需额外配置。

## 维护说明

- 新增页面：在 `pages/` 下创建新 HTML 文件，并更新 `index.html` 中的链接（如存在）。
- 修改页面：直接编辑对应文件，注意保持相对路径引用正确。
- 删除页面：移除文件及 `index.html` 中的相关链接。
- 资源更新：如需替换或新增公共资源，请放入 `assets/` 对应子目录，并检查所有引用页面。
- 提交规范：建议使用简洁的提交信息，例如“新增页面：xxx”、“修复页面 xxx 中的链接错误”。
